# 从 Carto 导出文件绘制地图的实用指南（Cities: Skylines 2）

你用 Carto 模组导出的文件夹里有 7 个矢量图层（GeoJSON + Shapefile）、4 个地形 GeoTIFF，以及 3 套 QGIS 样式主题（Plan / Street / Topo）。下面按"最推荐 → 备选 → 快速预览 → 关键坑"的顺序说明如何把它们画成一张地图。

---

## 1. 推荐主线：QGIS + 自带 .qml 样式（Plan / Street / Topo）

这是 Carto 作者唯一记录、且明确推荐的工作流。那 46 个 `.qml` 样式文件是为 QGIS 量身定制的，**只有 QGIS 能直接读懂它们**。QGIS 免费、开源，自带 GDAL/Python，不依赖你机器上的 geopandas/shapely，也不受系统里那个失效的 Microsoft Store Python 影响。

### 步骤

1. **安装 QGIS**：从官网下载长期支持版（LTR）独立安装包 `QGIS-OSGeo4W-3.44.11-1.msi`（[下载页](https://qgis.org/download/)），用默认选项安装即可。作者要求 3.28 "Firenze" 或更新版本；自带的 `.qml` 标记为 3.28.8，在 3.44 LTR 或最新 4.0.3 中都能正常加载（旧样式在新版 QGIS 中向前兼容，反过来才会丢样式）。**推荐装 3.44.11 LTR**，最稳。

2. **新建工程并设置坐标系**：`工程 (Project) > 新建`。点右下角 CRS 按钮（或 `工程 > 属性 > CRS`），选 **EPSG:32631（WGS 84 / UTM zone 31N）**，这样地图以"米"为单位，比例尺和距离才正确。

3. **加载 7 个矢量图层**：`图层 > 添加图层 > 添加矢量图层`（或直接拖拽），从 **`Shapefile\` 文件夹**（不是 GeoJSON）选入全部 7 个 `.shp`：Area_Boundary、Building_Boundary、Network_Boundary、Network_Centerline、POI_Location、Route_Centerline、Zoning_Boundary。Shapefile 已投影到 UTM（米），渲染才会按真实比例，不会被横向拉伸。

4. **加载地形栅格**：`图层 > 添加图层 > 添加栅格图层`，从 `GeoTIFF\` 选入 **Elevation.tif**（可玩区域高程，3.5 m）和 **Depth.tif**（水深，7 m）。WorldElevation/WorldDepth 是可玩区外的更大范围地形，默认样式不用，可不加。

5. **调整图层叠放顺序**（在"图层"面板拖动，顶部 = 最后绘制 = 在最上层）。按作者规定的顺序，从上到下：
   `POI_Location → Area_Boundary → Network_Centerline → Route_Centerline → Network_Boundary → Building_Boundary → Zoning_Boundary → Depth → Elevation`

6. **逐图层套用同一主题的 .qml 样式**：右键某图层 `> 属性 (Properties) > 符号化 (Symbology) 标签页 > 左下角"样式 (Style)"按钮 > 加载样式 (Load Style…)`，到 `Styles\<Plan|Street|Topo>\` 里选对应该图层的文件。**整张图只用一个主题**，不要混用（Plan = 白底高对比规划风；Street = 彩色详细日常风；Topo = 米色低饱和底图风）。
   - 加载前，确保"类别 (Categories)"区里 **Symbology 和 Labels 两个勾都打上**（栅格只有 Symbology）。
   - 文件名规则是 `<主题>-<要素>-<几何>.qml`：标注脚本选 `(Latin)`（英文）或 `(Chinese)`（中文）；路线/分区可选 `(By Color)` / `(By Transport)` / `(By Zoning)` 变体。

7. **地形上色与山体阴影**：把 `Styles\<主题>\<主题>-Terrain-Elevation.qml` 套到 Elevation.tif、`-Water-Depth.qml` 套到 Depth.tif。想要立体阴影，用 **Topo** 主题——它独有 `Topo-Terrain-Elevation-Hillshade.qml` 和 `-Contours.qml`。
   - 进阶真实山体阴影：`栅格 > 分析 > 山体阴影 (Hillshade)`，输入 Elevation.tif，参数用默认（Z 系数 1，方位角 300°，垂直角 40°）。把生成的灰度阴影放到彩色高程层下方，再把高程层混合模式设为"正片叠底 (Multiply)"或调到约 50% 不透明度。

8. **导出图片**：
   - 快速：`工程 > 导入/导出 > 导出地图为图片 (Export Map to Image)`，设 DPI、确认范围，存为 PNG/TIFF。
   - 高质量：`工程 > 新建打印布局 (Print Layout)`，用"添加地图"放入地图，再加比例尺（单位选米）、图例、指北针、标题，然后 `布局 > 导出为图片`（设 300 DPI、开抗锯齿，PNG/TIFF）或"导出为 PDF"（矢量输出）。

## 2. 备选：Python（matplotlib / rasterio）

适合不想装 QGIS、或想编程批量出图的情况。注意几点：

- **你的机器上没有 geopandas/shapely/fiona/pyogrio**，但 **rasterio 1.5.0、matplotlib 3.10.8、numpy、pandas、pillow 都已装好**，足以出图。
- **不要用 PATH 里的 `python`**——它是 Microsoft Store 的占位 stub，会静默退出（code 49）。用 `py -3.14` 或完整路径 `C:\Users\Yuanhao Lyu\AppData\Local\Python\pythoncore-3.14-64\python.exe`。
- **想装 geopandas 是可行的**：`py -3.14 -m pip install geopandas` 会成功（它拉取 shapely 2.1.2 + pyproj 3.7.2 + pyogrio 0.12.1，都有 cp314 轮子）。**不要 `pip install fiona`**——Python 3.14 没有 cp314 轮子，会触发源码编译并失败。geopandas 1.x 默认用 pyogrio，不需要 fiona。
- **`.qml` 样式无法被 matplotlib 读取**，只能当作配色/线宽/标注的参考，手动在代码里复刻。

### 不装 geopandas 也能画矢量（仅用标准库 json + matplotlib）

```python
import json
from matplotlib import pyplot as plt
from matplotlib.collections import PolyCollection, LineCollection

# 注意：GeoJSON 带 UTF-8 BOM，必须用 utf-8-sig
data = json.load(open(r"...\GeoJSON\Area_Boundary.json", encoding="utf-8-sig"))

polys = []
for f in data["features"]:
    for ring in f["geometry"]["coordinates"]:
        # 坐标是三维 [x, y, z]，只取前两维
        polys.append([(p[0], p[1]) for p in ring])

fig, ax = plt.subplots()
ax.add_collection(PolyCollection(polys, facecolor="#eee", edgecolor="#666", linewidths=0.3))
ax.set_aspect("equal"); ax.autoscale()
fig.savefig("map.png", dpi=200)
```
（线状图层 Network/Route_Centerline 用 `LineCollection` 同理。）

> 已经帮你写好的 `render_map.py` 就是这套思路的完整版（街道图 + 地形晕渲 + 分区彩图）。

### 地形山体阴影（rasterio + matplotlib）

```python
import rasterio, numpy as np
from matplotlib.colors import LightSource
import matplotlib.pyplot as plt

with rasterio.open(r"...\GeoTIFF\Elevation.tif") as src:  # EPSG:32631, int16, nodata -32768
    dem = src.read(1).astype(float)
    dem[dem == src.nodata] = np.nan
    ext = [src.bounds.left, src.bounds.right, src.bounds.bottom, src.bounds.top]

shade = LightSource(azdeg=315, altdeg=45).hillshade(dem, dx=3.5, dy=3.5, vert_exag=2)
plt.imshow(shade, cmap="gray", extent=ext); plt.gca().set_aspect("equal")
plt.savefig("terrain.png", dpi=200)
```

> 想把矢量叠在地形上时，**改用 `Shapefile\`（米）而非 GeoJSON（度）**，否则坐标系不一致、对不齐（详见第 4 节）。

## 3. 快速预览：浏览器 / 轻量工具

只想快速看一眼某个图层时：

- **小文件可直接拖进 [geojson.io](https://geojson.io) 或 VS Code 等在线/本地 GeoJSON 查看器**。能直接看的小图层：**Area_Boundary（约 616 KB）、POI_Location（约 5 MB）**。
- **超大文件不要直接打开**：Zoning_Boundary.json 约 233 MB、Network_Boundary 约 139 MB、Network_Centerline 约 96 MB——浏览器/在线工具会卡死或爆内存。对这些图层请改用对应的 Shapefile（在 QGIS 里加载，或 Python 用 pyogrio 增量读取），或用 `ijson` 流式解析。
- 地形 `.tif` 想快速看：用第 2 节的 rasterio + matplotlib 几行代码出张灰度/阴影图即可。

## 4. 关键提示 / 坑

- **坐标系（CRS）有两套，互不通用**：`Shapefile\` 和所有 GeoTIFF 都是 **EPSG:32631（UTM 31N，米）**，彼此对齐；`GeoJSON\` 是 **EPSG:4326（度）**。出正式地图、叠加栅格、量距离一律用 Shapefile（米）；用 GeoJSON（度）会让城市横向拉伸、比例尺失真。
- **文件大小**：上面列的几个大 GeoJSON 加载慢、占内存大。优先用 Shapefile 等价文件；QGIS 卡顿时可临时关掉重图层或启用渲染简化。
- **`.qml` 只服务 QGIS**：它们是 QGIS 专用 XML（标记 3.28.8 "Firenze"），用了规则渲染、点聚类、**随比例尺显隐**等特性，matplotlib/rasterio 无法解析。在 QGIS 里某些 POI/道路符号只在特定缩放级别才出现，全图缩放下未必都显示。转换工具只能转成 GeoServer SLD 且会丢精度——所以非 QGIS 路线只能手动复刻样式。
- **样式文件名 ≠ 图层名**（如 `Plan-Area-Boundary (Latin).qml` 对应图层 `Area_Boundary`），所以 QGIS 不会自动按名套用，批量样式插件也匹配不上——按教程**逐图层手动加载**。
- **各主题文件夹并不对称**：Building/Network-Boundary 无 Latin/Chinese 变体；Plan 主题没有 POI 样式；只有 Topo 有 Hillshade 和 Contours。按你选的主题里**实际存在的文件**去匹配。
- **字体**：中文样式用 Microsoft JhengHei UI、英文样式用 Segoe UI。你在 Windows 11 上 Segoe UI 自带，正常显示。
- **别用 World* 栅格当默认底**：WorldElevation/WorldDepth 是可玩区外的地形，套默认样式会把范围/比例拉偏；城市图请用 Elevation.tif / Depth.tif。
- **别浪费时间装 conda 或降级到 Python 3.12**：你这台 3.14 机器直接 `pip install geopandas`（走 pyogrio）就行，无需折腾。
