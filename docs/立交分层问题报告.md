# 2D 城市地图渲染中"立交分层 vs 匝道连续"的两难问题

> 一份用于咨询 GIS / 制图 / 计算几何专家的技术报告
> 场景:用 Python(matplotlib)把 Cities: Skylines 2 导出的路网数据渲染成经典城市地图

---

## 1. 目标

把路网渲染成一张"经典城市地图",道路部分需同时满足两条:

- **目标 A —— 交叉处区分上下(立交/上跨)**:高架、桥梁应明显压在被它跨越的道路之上,体现立体交通(下方道路在交叉处让出缺口,如同从桥下穿过)。
- **目标 B —— 连续道路不被切断**:同一条道路(尤其是**逐渐爬升的匝道**)在视觉上必须是连续的折线,不能出现莫名其妙的缺口/碎裂。

实践中发现 A 与 B 在现有数据与常规渲染机制下**相互冲突**,这正是本报告要请教的核心。

---

## 2. 可用数据

路网来自游戏 mod「Carto」导出的 GeoJSON,图层 `Network_Centerline`,共 **21089 条线要素(LineString)**。每条要素的关键属性:

| 属性 | 取值 | 说明 |
|---|---|---|
| `Object` | Road / Track / Pathway / Waterway / Runway / Taxiway | 道路 14818 条 |
| `Category` | Highway / Large / Medium / Small(及 Train/Subway/Tram…) | 道路等级 |
| `Form` | **Normal / Elevated / Tunnel** | 相对地面的形态(地面/高架/隧道) |
| `Elevation` | 浮点数(米) | **绝对**高程,**含地形起伏**,**每要素仅一个值** |
| 几何 | 折线顶点 (lon, lat) | 要素在路口处被切分(路口=要素端点) |

道路按 `Form` 分布(仅 Object=Road):**Normal 9038 / Elevated 5780 / Tunnel 781**。
高程:Normal 路均值 ≈114m(贴合地形),Elevated 路离地高度(Elevation 减地形)中位数 ≈21m。

**关键限制**:
1. 没有 OSM 那种人工标注的整数 `layer` / `level` 标签。
2. `Elevation` 是**每要素一个绝对值**(不是逐顶点),且把地形高度也算进去了。
3. 相连要素(同一条路被路口切分成的多段)在共享端点处坐标基本一致,但跨不同要素的连通性需自己用端点判断。

---

## 3. 渲染机制,以及"切断"是怎么产生的

道路画法(经典做法):每条路画**两道**——

1. 较**宽**的"描边 (casing)"在下(深色);
2. 较**窄**的"填充 (fill)"在上(路面色)。

二者叠加形成"带深色边的路面"。多条路相交/相接时,靠 **z 轴顺序 (zorder)** 决定谁盖谁。

> **"切断"的根源**:当 A 段路用**更高 zorder 的描边**盖住相邻/相连 B 段路的**填充端头**时,接缝处露出 A 的描边色,形成一个缺口——看起来 B 被切断了。

**同层无切断的经典解法**:**全局两遍**——先画**所有**描边,再画**所有**填充——于是所有填充都在所有描边之上,同层道路接缝处填充相连、无缝。**但这要求所有路"同层"。**

---

## 4. 核心两难(本报告的关键)

- **要满足 A(分上下)**:必须让"上方的路"用更高 zorder **盖住**"下方的路"——即在交叉处**主动制造遮挡/缺口**。
- **要满足 B(不切断)**:相连的路必须无缝——即**绝不能**互相盖断。

**矛盾**:仅凭高程 / zorder 排序,**无法区分"交叉(应盖断)"与"相连(不应盖断)"**——在 2D 投影上二者都是"几何上靠在一起"。

- **爬坡匝道**:相连的要素高程**不同**(逐渐升高)→ 高的盖断低的 → 沿匝道**处处被切**。
- **立交跨越**:相交的要素高程**不同** → 高的盖断低的 → **正确上跨**。
- 同一个"高程差导致的遮挡",对**相连**的是 bug(切碎),对**相交**的是 feature(上跨)。

---

## 5. 已尝试的方案及结果(踩坑全记录)

### (a) 逐要素连续高程 zorder
- **原理**:每要素 `casing 的 zorder = 高程`,`fill 的 zorder = 高程 + ε`。ε 既是描边-填充的间隙,也同时是**"上跨阈值"与"切断容忍度"**(数学上二者相等:相邻要素高程差 > ε 才会互相盖断;交叉要素高程差 > ε 才会分上下)。
- **效果**:立交上跨正确。
- **问题**:相连匝道相邻要素高程差 > ε 处被切。**ε 调大** → 减少切断,但也抬高上跨阈值(净空 < ε 的立交不分上下);**ε 调小** → 切断更多。**单一参数无法同时满足**。实测"切割很多"。

### (b) 离散分箱(按高程每 6m 一层,逐层绘制)
- **问题**:箱界处硬切;一条爬坡匝道跨越多个箱 → 多处接缝、明显"分段"。

### (c) 连通分量(并查集)
- **原理**:把**首尾相连**的高架要素并为一座"结构"(一桥/一组匝道),整座**共享一个高度层** → 结构内部无缝;不同结构按高度互跨。
- **效果**:内部切断大减。
- **问题**:① 结构与地面的过渡处仍切;② 端点坐标精度导致分量**碎裂**(本应连通的被判为两座结构)→ 仍有较明显接缝。

### (d) 完全摊平(所有路同层,全局两遍)
- **效果**:**零切断**,非常干净。
- **问题**:**完全不分上下**——所有交叉都平面相接,立交完全看不出上跨。

### (e) 逐交叉点桥盖(高计算量)
- **原理**:底图先**摊平**(零切断);再用空间网格**两两检测所有"严格交于两条线段各自内部"的交叉**——这种交叉必为立交(平面路口会共享端点,不会交在线段内部);对每个这种交叉,在**高程较高**的那条路上、交叉点处架一段**短"桥盖"**压在最上,从而盖住下方路 = 上跨。
- **效果**:理论正确——平面相接(共享节点)完全不动,只在真 X 形交叉叠加桥盖,不破坏相连道路。检测出约 1199 处交叉。
- **问题**:**在密集立交区,一条匝道会被它下穿的每一条上方道路各盖一个缺口 → 匝道被切得粉碎**(用户原话"史无前例地碎")。
- **根因**:**缺少"整条匝道属于同一层"的一致性**——每个交叉点**独立**判断上下,导致同一条下层匝道被反复打洞。

### (f) Form 三层(当前采用)
- **原理**:按 `Form` 把道路**一致地**归入三层——**隧道(最下,虚线) / 地面 / 高架(最上)**;每层内部用全局两遍 → 层内无缝;高架层**整体**压在地面层之上。
- **效果**:
  - 高架整层在上 → 跨地面路 = **干净上跨**;且**高架自身连续,绝不被下方路切碎**(它在上面,不会被下面的东西盖断)。
  - 地面路仅在**被高架跨越处**让出缺口(每条路通常一两处,干净)。
  - 同层相连匝道无缝。
  - **爬坡匝道**:其**地面段**在地面层、**高架段**在高架层,各自层内无缝;只在"上地面 → 上高架"的**过渡点**有一次衔接(同色相接,基本无感)。
- **残留局限**:
  1. **高架跨高架**(叠层立交、双层环路)同属高架层 → 并为一层,**不分上下**(但**不碎**)。
  2. 地面↔高架过渡点理论上有一次层级切换(实际同色相接,基本看不出)。

---

## 6. 当前结论

采用 (f) Form 三层,在"分上下"与"不碎"间取得目前最好的平衡:**主干立交(高架跨地面)干净上跨、匝道不碎**;代价是**高架互跨不再分上下**。本质上是用"层级是道路的属性(按 Form 一致赋予)"替代了"层级是交叉点的属性(逐点判断)",从而消除了碎裂。

2. 能否由几何/拓扑**自动推断出 OSM 式的整数 `layer`**(使相连道路共层、相交道路按相对高度分层)?例如:构建路网图,以"交叉点是否共享节点"为相连约束、以高程差为相对高度约束,做**约束传播 / 图着色 / 分层求解**?这类问题有没有成熟提法?

3. (e) 逐交叉点法的"过度遮挡碎裂",能否通过让**层级成为"整条 polyline / 连通路径"的属性(而非交叉点的属性)**来消除——即先给每条路定一个一致的层号,再渲染?在存在**环**(匝道自交、立交环线)时,如何保证层号自洽(可能无法用单一整数层满足所有约束,即"非平面"情形)?

5. 是否值得用 `Elevation - 地形高程 = 离地高度` 配合**短程一致性**(沿同一条路平滑)来重建逐顶点高度,从而更稳健地推断层级?代价与收益如何?
